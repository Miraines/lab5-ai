# Рефлексия

## Что изменилось за 3 практики

За время курса стало очевидно, что AI-инструменты — это не замена разработчику, а усилитель. Ключевой сдвиг: от "написать код руками" к "подготовить контекст и делегировать генерацию". Качество результата напрямую зависит от качества подготовки — спецификации, acceptance criteria, правил в CLAUDE.md.

Главный инсайт: 80% работы — это подготовка (spec, DoD, структура), 20% — собственно генерация и мониторинг.

## SDD vs ad-hoc — когда что

**SDD подходит**, когда:
- Фича имеет чёткие границы и acceptance criteria
- Нужен воспроизводимый процесс (другой разработчик / агент должен повторить)
- Проект долгоживущий, важна поддерживаемость

**Ad-hoc подходит**, когда:
- Быстрый прототип / proof of concept
- Одноразовый скрипт
- Исследовательская задача без чётких требований

## Какие инструменты буду использовать

- **Claude Code** — основной AI-ассистент для работы с кодом, спецификациями и рефакторингом
- **SDD-подход** — для структурированной разработки новых фич
- **n8n** — автоматизация рутинных процессов (AI code review, нотификации)
- **CodeRabbit** — автоматический AI code review в PR
- **CLAUDE.md / rules.md** — формализация правил проекта для агентов

# URL Shortener — Спецификация

## Что строим
Минималистичный HTTP-сервис на Go для сокращения URL-адресов.

## API

### POST /shorten
- Принимает JSON: `{"url": "https://example.com/very/long/path"}`
- Валидирует URL (scheme + host обязательны)
- Генерирует уникальный 6-символьный код (a-zA-Z0-9)
- Возвращает JSON: `{"short_url": "http://localhost:8080/abc123"}`
- Повторный запрос с тем же URL возвращает тот же код (идемпотентность)

### GET /{code}
- Редирект 301 на оригинальный URL
- Инкрементирует счётчик переходов
- 404 если код не найден

### GET /stats/{code}
- Возвращает JSON: `{"url": "https://example.com/...", "visits": 42}`
- 404 если код не найден

## Чего НЕ строим
- Персистентное хранилище (БД, файлы) — только in-memory
- Авторизация / rate limiting
- Кастомные короткие коды
- TTL / истечение ссылок
- UI / фронтенд

## Acceptance Criteria
1. `POST /shorten` с валидным URL возвращает 201 и JSON с `short_url`
2. `POST /shorten` с невалидным URL возвращает 400
3. `GET /{code}` с существующим кодом возвращает 301 редирект
4. `GET /{code}` с несуществующим кодом возвращает 404
5. `GET /stats/{code}` возвращает корректный счётчик визитов
6. Повторный `POST /shorten` с тем же URL возвращает тот же код
7. Все тесты проходят: `go test ./...`

## Edge Cases
- Пустое тело запроса → 400
- URL без scheme (`example.com`) → 400
- Пустой URL (`{"url": ""}`) → 400
- Коллизии кодов — перегенерация при совпадении

## Стек
- Go 1.21+, стандартная библиотека (net/http)
- Хранение: sync.RWMutex + map
- Тесты: testing + net/http/httptest
